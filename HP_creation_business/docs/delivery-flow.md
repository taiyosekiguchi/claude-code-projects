# 納品フロー

## 全体フロー（両プラン共通）

```
1. 受注（ココナラ）
   ↓
2. ヒアリング（Googleフォーム送付）
   ↓
3. 顧客が情報入力（24時間以内）
   ↓
4. 制作開始
   ↓
5. 仮完成・確認依頼
   ↓
6. 修正対応（1回まで無料）
   ↓
7. 納品
   ↓
8. 納品完了・評価依頼
```

**所要時間**: 3〜5営業日

---

## スタンダードプランの納品フロー

### Day 1: 受注・ヒアリング

#### 1-1. 受注確認
- ココナラで受注通知
- 顧客情報を確認
- Notionに案件登録

#### 1-2. 初回連絡（テンプレート使用）
```
件名: 【企業HP制作】ヒアリングのお願い

〇〇様

この度はご依頼いただきありがとうございます。
早速制作を進めさせていただきます。

以下のフォームから必要な情報をご入力ください。
https://forms.google.com/xxxxx

ご入力後、2-3営業日で制作を完了いたします。

よろしくお願いいたします。
```

**作業時間**: 5分

### Day 2-3: 制作

#### 2-1. 情報受領確認
- Googleフォームの回答を確認
- company-data.jsonに転記

#### 2-2. 制作作業
```bash
# テンプレートから作成
git clone corporate-site-template tanaka-corp
cd tanaka-corp

# 情報入力
vi config/company-data.json

# プレビュー確認
npm run dev

# ビルド
npm run package
```

**作業時間**: 30〜60分

#### 2-3. 仮完成連絡
```
件名: 【仮完成】ご確認をお願いいたします

〇〇様

企業ホームページが仮完成いたしました。
以下のURLでプレビューをご確認ください。

プレビューURL: https://preview.vercel.app/xxxxx

ご確認いただき、修正箇所がございましたら
ご連絡ください（1回まで無料で対応いたします）。

問題なければ、そのまま納品に進めさせていただきます。

よろしくお願いいたします。
```

**作業時間**: 5分

### Day 4: 修正対応（必要な場合）

#### 3-1. 修正内容確認
- 顧客からのフィードバック受領
- 修正内容をリスト化

#### 3-2. 修正作業
```bash
# 修正
vi config/company-data.json
# または components/xxx.tsx

# 再ビルド
npm run package
```

**作業時間**: 15〜30分

### Day 5: 納品

#### 4-1. 納品物準備
- [ ] site.zip（静的HTML一式）
- [ ] アップロード手順書.pdf
- [ ] サーバー比較表.pdf
- [ ] README.txt

#### 4-2. 納品メール送付

```
手順書に従ってサーバーにアップロードしてください。

■ 推奨サーバー
さくらのレンタルサーバ（スタンダード）
月額425円
https://www.sakura.ne.jp/

■ サポート期間
納品後1週間はメールサポートを無料で行います。
ご不明点がございましたらお気軽にお問い合わせください。

この度はありがとうございました。
公開後、ぜひURLをお知らせください！

よろしくお願いいたします。
```

**作業時間**: 10分

#### 4-3. ココナラで正式な納品

## おまかせプランの納品フロー

### Day 1: 受注・ヒアリング

スタンダードプランと同じ。

### Day 2-3: 制作・デプロイ

#### 2-1. 制作作業
```bash
# テンプレートから作成
git clone corporate-site-template tanaka-corp
cd tanaka-corp

# 情報入力
vi config/company-data.json

# Gitリポジトリ初期化
git init
git add .
git commit -m "feat: 株式会社田中商事 企業サイト"
git remote add origin https://github.com/YOUR/tanaka-corp.git
git push -u origin main

# Vercelにデプロイ
vercel --prod
```

**作業時間**: 30〜60分

#### 2-2. 仮公開連絡
```
件名: 【仮公開】ご確認をお願いいたします

〇〇様

企業ホームページが仮公開されました。
以下のURLでご確認ください。

仮公開URL: https://tanaka-corp.vercel.app

スマートフォンでもご確認いただけます。
修正箇所がございましたらご連絡ください。

問題なければ、独自ドメインでの公開に進めます。

よろしくお願いいたします。
```

**作業時間**: 5分

### Day 4: ドメイン設定

#### 3-1. カスタムドメイン追加
```bash
# Vercelにドメイン追加
vercel domains add tanaka-corp.com

# DNS情報取得
vercel domains inspect tanaka-corp.com
```

#### 3-2. DNS設定案内メール
```
件名: 【DNS設定のお願い】独自ドメインで公開します

〇〇様

独自ドメイン（tanaka-corp.com）での公開準備が整いました。

以下のDNS設定をお願いいたします。
設定は5分程度で完了します。

━━━━━━━━━━━━━━━━━━━━━━━━
【DNS設定内容】

お名前.comの管理画面で以下を設定:

レコードタイプ: A
ホスト名: @（または空欄）
値: 76.76.21.21
TTL: 3600

━━━━━━━━━━━━━━━━━━━━━━━━

【設定方法】
1. お名前.com Naviにログイン
   https://www.onamae.com/
2. 「DNS設定」メニューをクリック
3. 上記の内容を入力して保存

※設定方法の詳細は添付PDFをご確認ください。

分からない場合は、お名前.comサポート
（0120-865-209）に電話で相談できます。
「Aレコードを追加したい」とお伝えください。

設定完了後、ご連絡いただければ
24時間以内に独自ドメインでアクセスできるようになります。

よろしくお願いいたします。
```

**作業時間**: 10分

### Day 5: 納品完了

#### 4-1. 公開確認
- DNS設定反映を確認
- https://tanaka-corp.com でアクセス確認

#### 4-2. 納品完了メール
